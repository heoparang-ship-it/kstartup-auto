- **E3 (대구)**: region ∈ {전국, 대구, 경북, 영남} 또는 agency에 대구·경북 기관 → green. 인천 단독/서울 단독/기타 지역 → red.

## 티어 결정 로직 (v4)

```
1. DOMAIN_NEGATIVE_HARD 2개 이상 hit → 무조건 RED (엔티티 무관)
2. LIFECYCLE 레이어 0개 + 범용 L0 0개 → 엔티티별 평가 건너뛰고 RED
3. 엔티티별 독립 평가:
   - 지역 미스매치 → 해당 엔티티 red
   - 자격 실격 → 해당 엔티티 red
   - 자격 green + 라이프사이클 hit ≥ 2 → 해당 엔티티 green
   - 자격 green + 라이프사이클 hit = 1 → 해당 엔티티 yellow
   - 자격 yellow → 해당 엔티티 yellow (지역 모호 또는 자격 부분 충족)
4. best_entity = 3개 중 tier가 가장 높은 엔티티 (green > yellow > red)
5. best_tier = best_entity의 tier
6. 최종 표시 tier = best_tier
```

## Sonnet Verdict 프롬프트 (deep verdict용)

🟢 전체 + 🟡 상위 10건은 Sonnet이 공고 원문/PDF를 읽고 4개 질문에 답변:

1. **생애주기 레이어 매칭**: 이 공고가 L1~L7 중 어느 레이어와 상품으로 연결되는가? (없으면 "무관")
2. **도메인 네거티브 체크**: 도시안전/제조/의료기기/TRL 증빙 등에 해당하는가?
3. **엔티티별 자격**: E1(허파랑 개인)·E2(XCom 법인)·E3(이영림 대구) 각각 자격 요건 충족하는가?
4. **추천 엔티티**: 지원 시 가장 유리한 엔티티는? 1줄 이유.

출력 스키마:
```json
{
  "layer_match": "L2_허니문|L3_콘텐츠|없음",
  "negative_flags": [],
  "entity_verdict": {
    "heoparang_personal": {"qualified": true, "reason": "..."},
    "xcom_corp":          {"qualified": true, "reason": "..."},
    "younglim_daegu":     {"qualified": false, "reason": "지역 미스매치"}
  },
  "recommended_entity": "xcom_corp",
  "rationale": "인천 거점 + AI 엔진 포지션으로 매칭 최적"
}
```

## 용어 일치

- "E1/E2/E3" = 엔티티 3종 코드 (허파랑 개인 / XCom 법인 / 이영림 대구)
- "L1~L7" = 생애주기 레이어 (결혼준비·허니문·콘텐츠·이벤트·육아·주거·금융)
- "L0" = 범용 B2C 플랫폼 키워드
- "domain.whitelisted" = 라이프사이클 or L0 레이어 1개 이상 hit
- "best_entity / best_tier" = 3개 엔티티 중 최고 tier 엔티티
- "audit_flags" = green → yellow 강등 사유 배열
- "negative_flags" = DOMAIN_NEGATIVE_HARD 히트 카테고리
- "nogo_history" = 사용자 NO-GO 누적 기록
